Conversations - Content Submission

Use the Bazaarvoice Mobile SDKs to enable Conversations functionality, such as Ratings and Reviews. The Conversations module provides an easy-to-use wrapper around the Conversations API. Contact Bazaarvoice to set up Conversations.

  1. Review Submission
  2. Review Comment Submission
  3. Question Submission
  4. Answer Submission
  5. Feedback Submission
  6. Form Parsing

  Important: You will need your apiKeyConversations to implement and use Conversations with Mobile SDKs.


Installation Instructions

Click here to see how to install and configure the BVSDK.

Installation with Gradle

Include the Maven Central repository and add the appropriate Bazaarvoice Mobile SDK modules to your dependencies:

dependencies {
    compile 'com.bazaarvoice.bvandroidsdk:conversations:6.+'
    compile 'com.google.android.gms:play-services-ads:9.{your_play_services_version}'
    compile 'com.android.support:recyclerview-v7:25.{your_support_library_version}'
}
repositories {
    mavenCentral()
}

Extending the Application

Create a class that extends android.app.Application, and initialize the Mobile SDK using its builder.

You can find a full list of build options in BVSDK.java.

in the following example, YOUR_CLIENT_ID represents your client name, such as "bazaarvoice". If you are unsure about your client ID or API key values, contact Client Care or your implementation team.

public class MyApplication extends Application {

    @Override
    public void onCreate(){
        super.onCreate();
        BVSDK.builder(application, BazaarEnvironment.PRODUCTION)
          .logLevel(BVLogLevel.VERBOSE) // Optional: Set the log level. Default level is ERROR.
          .dryRunAnalytics(false)       // Optional: Set analytics to log only, not make a network request. Default is false.
          .okHttpClient(okHttpClient)   // Optional: Use you own okHttpClient instance
          .build();
    }
}

AndroidManifest

<!-- set the name of the class for the application entry -->
<application
    android:name=".MyApplication">
</application>

Using Conversations

After you install the Mobile SDK, complete the following steps to submit consumer-generated content.


BVConversationsClient

Use an instance of the BVConversationsClient to make all your Conversations Submission API requets. You can keep an instance of this this object to handle all your Conversations requests as shown in the examples below.

  • ReviewSubmission
  • ReviewCommentSubmission
  • QuestionSubmission
  • AnswerSubmission
  • FeedbackSubmission

Submitting Reviews

To submit content, form the proper ConversationsSubmissionRequest subclass, and send it through the BVConversationsClient. 

ReviewSubmissionRequest previewSubmission = new ReviewSubmissionRequest
  .Builder(Action.Preview, productId)
  .userNickname(userNickname)
  .addVideoUrl(videoUrl, optionalVideoCaption) // add youtube video link, if your configuration supports it
  .build();

BVConversationsClient client = new BVConversationsClient.Builder(BVSDK.getInstance()).build(); // Use one instance
client.prepareCall(previewSubmission).loadAsync(new ConversationsCallback<ReviewSubmissionResponse>() {
  @Override
  public void onSuccess(ReviewSubmissionResponse response) {
    FormData formData = response.getFormData(); // To view required/optional form fields
    boolean hasErrors = response.getHasErrors(); // If form data was not sent correctly, or some other backend processing issues exist
    FormError formErrors = response.getFormErrors(); // Object containing the form errors
    List<Error> genericBvErrors = response.getErrors(); // Object containing the other backend processing issues

    // If hasErrors is false, then the submission succeeded
    // Make sure to send with Action.SUBMIT for it to fully complete
  }
  @Override
  public void onFailure(BazaarException exception) {
    exception.printStackTrace();
  }
});

Parsing Form Data

Form Data can be accessed in the onSuccess callback for any ConversationsSubmissionRequest with Action.Preview. It will contain a list of all the form fields, with the keys that you need for a request, and a boolean for whether it is a required field or not, as well as other important metadata.

For more information on how the API processes Form Data please see: Submission Fundamentals: Request Form Information 

BVConversationsClient client = new BVConversationsClient.Builder(BVSDK.getInstance()).build(); // Use one instance
client.prepareCall(previewSubmission).loadAsync(new ConversationsCallback<>() {
  @Override
  public void onSuccess( response) {
    FormData formData = response.getFormData(); // To view required/optional form fields
    Map<String, FormField> formFieldMap = formData.getFormFieldMap();
    for (Map.Entry<String, FormField> fieldEntry : formFieldMap.entrySet()) {
      String key = fieldEntry.getKey(); // Name of the key that must be used for submission
      FormField formField = fieldEntry.getValue(); // Field containing metadata
      boolean isRequired = formField.isRequired(); // Whether this field is required or optional
    }
  }

  @Override
  public void onFailure(BazaarException exception) {
    exception.printStackTrace();
  }
});
