Targeted Advertising

The Advertising module of the Bazaarvoice Mobile SDKs is powered by Bazaarvoice's powerful shopper segment data and delivers highly targeted ads to your users. You must have Shopper Advertising enabled to use the Advertising module.

Bazaarvoice creates custom ad markup using DoubleClick for Publishers (DFP) based on your specifications. You can use the Bazaarvoice Mobile SDKs to easily pair the DFP SDK with the Advertising module. Contact Bazaarvoice to set up targeted ads in DFP.

  Important: You will need your apiKeyShopperAdvertising and DFP adUnitIds to implement and use the Advertising module.


Installation Instructions

Click here to see how to install and configure the BVSDK.

Installation with Gradle

Include the Maven Central repository and add the appropriate Bazaarvoice Mobile SDK modules to your dependencies:

dependencies {
    compile 'com.bazaarvoice.bvandroidsdk:advertising:4.+'
    compile 'com.google.android.gms:play-services-ads:'
}
repositories {
    mavenCentral()
}

Extending the Application

Create a class that extends android.app.Application, and initialize the Mobile SDK using its builder.

You can find a full list of build options in BVSDK.java.

in the following example, YOUR_CLIENT_ID represents your client name, such as "bazaarvoice". If you are unsure about your client ID or API key values, contact Client Care or your implementation team.

public class MyApplication extends Application {

    @Override
    public void onCreate(){
        super.onCreate();

        // Builder used to initialize the Bazaarvoice SDKs
        BVSDK bvsdk = new BVSDK.Builder(this, YOUR_CLIENT_ID) //your client name, e.g. 'bazaarvoice'.
            .bazaarEnvironment(BazaarEnvironment.PRODUCTION) // production or staging
            .apiKeyShopperAdvertising(YOUR_SHOPPER_ADVERTISING_API_KEY) //Required for Ads
            .logLevel(BVLogLevel.INFO)
            .build();
    }
}

AndroidManifest

<!-- set the name of the class for the application entry -->
<application
    android:name=".MyApplication">
</application>

Set User Information

After a user is authenticated inside your user login workflow, set an authenticated user to enable profile matching. Refer to User Authentication Token for instructions.

  Note: You must give the userAuthString created with your existing Bazaarvoice implementation logic to the BVSDK.getInstance() object. However, the steps to encrypt your userAuthString are applicable even if you are not a Conversations client.

Implementing Ads

Bazaarvoice recommends you implement advertisements using DFP's SDK for Android. Contact Bazaarvoice for assistance. After ads are implemented, you can add targeting data to your ads.

For every PublisherAdRequest.Builder you use, call the addCustomTargeting method with the following code:

...
PublisherAdRequest.Builder publisherAdRequest = new PublisherAdRequest.Builder();
Map targetingKeywords = BVAds.getCustomTargeting();
for (Map.Entry entry : targetingKeywords.entrySet()) {
    publisherAdRequest.addCustomTargeting(entry.getKey(), entry.getValue());
}
...

The above call sends DFP the information necessary to target ads correctly.


Upgrading from Older Versions of Bazaarvoice Mobile SDKs

In previous versions, the Mobile SDKs included the DFP SDK, and you needed to use methods like bvAds.getTargetedAdRequest() instead of new PublisherAdRequest.Builder(). However, Bazaarvoice has simplified this process greatly with the latest Mobile SDKs. If you want to use the Advertising module of the most up-to-date Mobile SDKs, you must implement DFP ads yourself. Google provides excellent documentation to guide you through the process. After ads are implemented and working well, use the call in the "Implementing Ads" section to add targeting information.


Complete the following steps to finish upgrading your Mobile SDK:

  • Change getTargetedAdLoader(context, adUnitId) to new AdLoader.Builder(context, adUnitId)
  • Change getTargetedAdRequest() to new PublisherAdRequest.Builder()
  • Change getTargetedInterstitialAd(context, adUnitId) to PublisherInterstitialAd interstitialAd = new PublisherInterstitialAd(context); interstitialAd.setAdUnitId(adUnitId);
  • Change getTargetedAdView(context, adUnitId) to PublisherAdView bannerAd = new PublisherAdView(context); bannerAd.setAdUnitId(adUnitId);

Determining Contextual Interests

The Advertising module is structured around product categories. You can find a complete list in BVContextualInterests.java. The list is split into two tiers, with about 30 Tier One categories, each of which includes multiple Tier Two subcategories.

For example, the Women's Dresses section of a department store would correspond to the following product categories:

  • Tier One: tierOneApparelAndAccessories
  • Tier Two: tierTwoApparelAndAccessoriesWomensDresses

Similarly, the Television section of an electronics store would correspond to the following product categories:

  • Tier One: tierOneConsumerElectronics
  • Tier Two: tierTwoConsumerElectronicsTelevisions

All Tier Two categories are extensions of their Tier One category names.

Contextual interest categories are used while sending location information to the Advertising module, as defined in the following section.


Notes on Advertising and User Location

The Advertising module provides many ways to integrate location data into a shopper's profile. You can find all location-related methods in Ads.java. Bear in mind the following information:

  • All location methods have contextualTier1 and contextualTier2 parameters. This allows you to send all location->interest mapping information available to the Advertising module to give the shopper the best profile possible. Refer to BVContextualInterests.java for a generated list of Contextual Tier 1 and Tier 2 strings.
  • Bazaarvoice defines methods that correlate strongly to LocationListener interface methods. You can find example usage within LocationManagerExmaple.java.
  • Bazaarvoice also defines methods that correlate strongly to Gimbal PlaceEventListener interface methods. You can find example usage within GimbalExample.java.
