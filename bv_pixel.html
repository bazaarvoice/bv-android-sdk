BV Pixel

The BV Pixel (formerly known as the ROI beacon) tells Bazaarvoice when a customer converts in your Android app. There are two types of BV Pixel tags:

  • Generic (non-commerce) conversion tags capture events such as newsletter signups, product registration, downloads of a PDF, and requests for more information about a product or service. You can generate ROI (return on investment) reports related to key performance indicators.
  • Transaction conversion tags capture purchase data—when a customer buys a product. The data captured might include SKUs, email addresses, and postal addresses. You can generate ROI reports tied to revenue.

After you implement the BV Pixel in your Android app, you can use this measurement dashboard to generate reports that show how users are interacting in your Android app, and to understand the value of your implementation. You can also enable other useful features. The maintenance-free post-interaction email (PIE) feature is key to increasing the volume of your consumer-generated content. In addition, you can run A/B tests to evaluate best practices in your Android app, live, before you implement them for all customers. You must install the BV Pixel before you can use these indispensable features.


Installation Instructions

Click here to see how to install and configure the BVSDK.

Installation with Gradle

Include the Maven Central repository and add the appropriate Bazaarvoice Mobile SDK modules to your dependencies:

dependencies {
    compile 'com.bazaarvoice.bvandroidsdk:common:5.+'
}
repositories {
    mavenCentral()
}

Extending the Application

Create a class that extends android.app.Application, and initialize the Mobile SDK using its builder.

You can find a full list of build options in BVSDK.java.

in the following example, YOUR_CLIENT_ID represents your client name, such as "bazaarvoice". If you are unsure about your client ID or API key values, contact Client Care or your implementation team.

public class MyApplication extends Application {

    @Override
    public void onCreate(){
        super.onCreate();

        // Builder used to initialize the Bazaarvoice SDKs
        BVSDK.builder(this, YOUR_CLIENT_ID) //your client name, e.g. 'bazaarvoice'.
            .bazaarEnvironment(BazaarEnvironment.PRODUCTION) // production or staging
            .logLevel(BVLogLevel.INFO)
            .build();
    }
}

AndroidManifest

<!-- set the name of the class for the application entry -->
<application
    android:name=".MyApplication">
</application>

Tracking transaction conversions

Implement the following code to use the BV Pixel to track transaction conversions.

/*
  Build a list of TransactionItem based on the transaction.
  Sku is required but the more parameters provided the better.
*/

TransactionItem firstItem = new TransactionItem.Builder("123")
        .price(10.99)
        .category("pets")
        .name("cat food")
        .imageUrl("http://example.com/catfood.jpg")
        .build();

TransactionItem secondItem = new TransactionItem.Builder("321")
        .price(14.99)
        .category("pets")
        .name("dog food")
        .imageUrl("http://example.com/dogfood.jpg")
        .build();

List items = new ArrayList<>();
items.add(firstItem)
items.add(secondItem);

/*
  Build a Transaction
  OrderId, orderTotal, and orderItems are required.
  Include any other key value params you want.
  BVTransaction also has convenience builders for commonly used params.
*/

Map<String, Object> params = new HashMap<>();
params.put("email", "some.one@domain.com");

Transaction transaction = new Transaction.Builder("order id", 33.11, items, params)
        .shipping(4.99)
        .tax(2.14)
        .build();

/*
  Use the Bazaarvoice Mobile SDK to track the Conversion Transaction Event.
*/

BVSDK.getInstance().sendConversionTransactionEvent(transaction);

Tracking generic non-commerce conversions

Implement the following code to use the BV Pixel to track generic non-commerce conversions.

/*
  Construct a Conversion
  Type and value are required. Label is recommended.
  Include any other key value params you want.
*/

Map<String, Object> params = new HashMap<>();
params.put("email", "some.one@domain.com");

Conversion conversion = new Conversion("BrochureDownload", "YourDefinedValue", "Model xyz brochure", params);

/*
  Use the BVSDK to track the Non-commerce Conversion Event.
*/

BVSDK.getInstance().sendNonCommerceConversionEvent(conversion);
