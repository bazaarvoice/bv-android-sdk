BV Pixel

What is it?

The BV Pixel tells Bazaarvoice when a customer converts in your Android app. There are two types of BV Pixel tags:

  • Generic (non-commerce) conversion tags capture events such as newsletter signups, product registration, downloads of a PDF, and requests for more information about a product or service. You can generate ROI reports related to key performance indicators.
  • Transaction conversion tags capture purchase data -- when a customer buys a product. The data captured might include SKUs, email addresses, and postal addresses. You can generate ROI reports tied to revenue. After you install the BV Pixel, you can use this measurement dashboard to generate reports that show how users are interacting in your Android app, and to understand the value of your implementation.

Why is it important?

After you implement the BV Pixel in your Android app, you can enable other useful features besides reporting. The maintenance-free post-interaction email feature is key to increasing the volume of your user-generated content. In addition, with the BV Pixel you can run A/B tests to evaluate best practices in your Android app, live, before you implement them for all customers. You must install the BV Pixel before you can use these indispensable features.

Installation with Gradle

Include the Maven Central repository and add Bazaarvoice's modules to your dependencies:

dependencies {
    compile 'com.bazaarvoice.bvandroidsdk:common:4.0.0'
}
repositories {
    mavenCentral()
}

Extend Application

Create a class that extends android.app.Application and initialize the SDK using its builder.

A full list of build options are found in BVSDK.java.

Below, YOUR_CLIENT_ID represents your client name, e.g. "bazaarvoice". If you are unsure of your client ID or API key values, you can request help through the support portal or your implementation team.

public class MyApplication extends Application {

    @Override
    public void onCreate(){
        super.onCreate();

        // Builder used to initialize the Bazaarvoice SDKs
        BVSDK bvsdk = new BVSDK.Builder(this, YOUR_CLIENT_ID) //your client name, e.g. 'bazaarvoice'.
            .bazaarEnvironment(BazaarEnvironment.PRODUCTION) // production or staging
            .logLevel(BVLogLevel.INFO)
            .build();
    }
}

AndroidManifest

<!-- Request internet permission, if not already there. -->
<uses-permission android:name="android.permission.INTERNET" />

<!-- set the name of the class for the application entry -->
<application
    android:name=".MyApplication">
</application>

Using BV Pixel to track transaction conversions:

/*
  Build a list of TransactionItem based on the transaction.
  Sku is required but the more parameters provided the better
*/

TransactionItem firstItem = new TransactionItem.Builder("123")
        .price(10.99)
        .category("pets")
        .name("cat food")
        .imageUrl("http://example.com/catfood.jpg")
        .build();

TransactionItem secondItem = new TransactionItem.Builder("321")
        .price(14.99)
        .category("pets")
        .name("dog food")
        .imageUrl("http://example.com/dogfood.jpg")
        .build();

List items = new ArrayList<>();
items.add(firstItem)
items.add(secondItem);

/*
  Build a Transaction
  OrderId, orderTotal, and orderItems are required
  You may include any other key value params you'd like
  BVTransaction also has convenience builders for commonly used params
*/

Map<String, Object> params = new HashMap<>();
params.put("email", "some.one@domain.com");

Transaction transaction = new Transaction.Builder("order id", 33.11, items, params)
        .shipping(4.99)
        .tax(2.14)
        .build();

/*
  Use the BVSDK to track the Conversion Transaction Event
*/

BVSDK.getInstance().sendConversionTransactionEvent(transaction);

Using BV Pixel to track generic non-commerce conversions:

/*
  Construct a Conversion
  Type and value are required. Label is recommended
  You may include any other key value params you'd like
*/

Map<String, Object> params = new HashMap<>();
params.put("email", "some.one@domain.com");

Conversion conversion = new Conversion("BroucherDownload", "YourDefinedValue", "Model xyz brochure", params);

/*
  Use the BVSDK to track the Non-commerce Conversion Event
*/

BVSDK.getInstance().sendNonCommerceConversionEvent(conversion);
