Authenticity using device fingerprinting

Note: Iovation is requied for US customers implementing UGC Content Submission with the Conversations API.

Bazaarvoice has partenered with iovation, an industry leader in device reputation technology, to ensure authenticity in consumer-generated content. Bazaarvoice Mobile SDKs for Android and iOS support the ability to include the iovation fingerprint along with Conversations API Submission requests, using the fingerPrint parameter. Contact your Bazaarvoice support representative to request an evaluation of the iovation integration.

Refer to the Device Fingerprinting page of the Bazaarvoice Developer Portal for technical details about creating and submitting device fingerprinting information from iovation with consumer-generated content. Refer to the Mobile Device Fingerprinting page of the Developer Portal for mobile-specific details about integrating with iovation.


Downloading the iovation SDK

If you are using Conversations API then you may access the iovation aar through ConfigHub. Log in in to workbench, then navigate to Settings > Manage Applications > Edit > Technical Setup > Mobile SDK.

If you are using PRR or aren't sure how to log into ConfigHub, drop us a note at Mobile Development at Bazaarvoice.

Android Integration

After you integrate the iovation aar, the rest of the integration process is simple. For any ReviewSubmission, QuestionSubmission, or AnswerSubmission object passed to the BVConversationsClient#prepareCall(...) method, just add the blackbox value to the fingerPrint parameter like the sampe code below: 

BVConversationsClient client = new BVConversationsClient();

String blackbox = getBlackbox(getContext().getApplicationContext());

ReviewSubmission submission = new ReviewSubmission.Builder(Action.Submit, productId)
	.fingerPrint(blackbox)
	// add other submission params here
	.build();

client.prepareCall(submission).loadAsync(new ConversationsCallback<ReviewSubmissionResponse>() {

    @Override
    public void onSuccess(ReviewSubmissionResponse response) {
        // Successful repsonse
    }

    @Override
    public void onFailure(BazaarException exception) {
        // Failure, handle error appropriately
    }
});
}

Sample Application

The demo app file DemoConversationsAPIFragment.java contains the necessary iovation SDK integration code commented out. After you add the iovation aar to the project., uncomment the code requesting and adding the blackbox parameter to your submission requets. Complete the following steps to run a demo:

  1. Open the demo project.
  2. Create a directory named libs in the bv-android-sdk/app/ directory.
  3. Drag and drop the deviceprint-release-2.3.0.aar into the libs directory.
  4. Uncomment the compile(name:'deviceprint-release-2.3.0', ext:'aar') line in the bv-android-sdk/app/build.gradle file.
  5. Uncomment the blackbox and fingerprinting sample lines in DemoConversationsAPIFragment#setUpUGCSubmisionActionButtons(View view).
  6. Uncomment the start(getApplicationContext()) call in DemoMainActivity.
  7. Configure the project with your Client ID and Conversations API Key in the DemoConstants class.

After you complete integration, run the app and view the ReviewSubmission (or Question/Answer Submission0 requests when tapping the Review button in the Content Submission section of the Conversations demo page, as shown in the following examples.
