FAQ & Support

Frequently Asked Questions


  • Upgrading from BVSDK V2

    If you were previously using version 2 of this SDK, it only supported the Conversations API. This is now contained as the conversations module, so minimal code change will be required. Follow the instructions listed for installation and configuration in the Conversations Guide and remove the V2 source code.

  • How do I check out older versions of the SDK?

    You can checkout the latest version, as well as older versions since v3.0.1 on Maven Central. We do recommend using the latest version though for the most up to date bug fixes, etc. Also, older versions of the BVSDK are tagged according to version number in the Github Repo. You can list available versions with git tag and switch to a previous release with git checkout [tag]

  • Upgrading Advertising

    Bazaarvoice has removed the need to use DFP bundled with this SDK, to make it easier to implement. For more information, follow the instructions listed for installation and configuration in the Advertising Guide.

  • Does the BVSDK Use the Android Advertising ID?

    Yes, the Android Advertising ID is used for both the Advertising and Recommendations modules. The BVSDK will check if the user has "Limit Ad Tracking" enabled, and respects it if enabled.
